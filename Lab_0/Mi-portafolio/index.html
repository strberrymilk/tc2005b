Preguntas

  1. ¿Cuál es la diferencia entre Internet y World Wide Web?
    2. El Internet (interconnected network) es una red global que conecta a millones de dispositivos en todo el mundo. La WWW (world Wide Web) es un sistema de información que contiene documentos de hipertexto interconectados a través de Internet. En otras palabras, la WWW es una parte del internet que permite acceder a páginas web mediante navegadores web, mientras que el internet es la infraestructura que la hace posible, al proveer de comunicación e intercambio de datos.

  2. ¿Cuáles son las partes de un URL?
    3. Un URL (Uniform Resource Locator) es la dirección única de un recurso en la Web. Las partes principales de un URL son:
    • Protocolo/esquema: Protocolo utilizado para acceder al recurso, como HTTPS o HTTP.
    • Dominio/host: Servidor web donde se aloja el recurso.
    • *Puerto: Puerto del servidor. Suele omitirse.
    • Ruta/path: Ubicación del recurso en el servidor.
    • *Parámetros: Pares de clave-valor que se envían al servidor.
    • *Ancla: Otra parte del recurso que se utiliza para navegar dentro de la página.
    • Ejemplo: https://www.example.com:80/path/to/myfile.html?key1=value1&key2=value2#SomewhereInTheDocument

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    4. Un método HTTP es la instrucción que indica qué acción quiere realizar el cliente sobre un recurso del servidor.
    • GET: Solicita datos de un recurso especificado.
    • HEAD: Solicita los encabezados de respuesta de un recurso, sin el cuerpo.
    • POST: Envía datos al servidor para crear o actualizar un recurso.
    • PUT: Actualiza un recurso existente.
    • PATCH: Realiza una actualización parcial de un recurso.
    • DELETE: Elimina un recurso especificado.

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    5. POST, porque este método permite enviar datos al servidor en el cuerpo y no en la URL. Esto es especialmente importante cuando se trata de información sensible, como el usuario o contraseña. Además, POST está pensado para cambiar el estado del servidor, por ejemplo, un inicio de sesión.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    6. GET, porque el navegador está solicitando datos de un recurso específico, sin enviar información adicional al servidor ni cambiar su estado.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    7. El código 200 significa que la solicitud se procesó correctamente y el servidor devolvió la respuesta esperada. No ocurrió ningún error.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    8. Sí, porque el código de error 404 indica que el recurso solicitado no se encontró en el servidor.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    9. Sí, porque el código de error 500 indica que hubo un problema interno en el servidor al procesar la solicitud.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    10. Un atributo HTML5 está depreciado o desaprobado cuando ya no se recomienda su uso en el desarrollo web, generalmente porque ha sido reemplazado por una alternativa más moderna o eficiente. Algunos elementos de HTML 4 que están desaprobados en HTML5 incluyen:
    • center: Se usaba para centrar contenido. Ahora se recomienda usar CSS para centrar elementos.
    • font: Se usaba para definir el estilo de fuente. Ahora se recomienda usar CSS para este propósito.
    • acronym: Se usaba para definir acrónimos. Ahora se recomienda usar el elemento o CSS.

  10. ¿Cuáles son las diferencias principales entre HTML4 y HTML5?
    11. HTML5 introdujo varias mejoras y nuevas características en comparación con HTML4, entre las que destacan:
    • Introduce nuevos elementos semánticos, como header, footer, section, entre otros.
    • Soporte nativo para multimedia con elementos como audio y video.
    • APIs para mejorar la interactividad, como geolocalización, almacenamiento local, entre otras.
    • Mejor soporte para gráficos y animaciones con el elemento canvas y la integración de SVG.
    • Formularios mejorados con nuevos tipos de entrada y atributos.
    • Doctype simplificado: !DOCTYPE html en lugar de la declaración larga y compleja de HTML4.

  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    12. La tabla tiene la etiqueta table, que es el contenedor principal. Dentro de la tabla, se pueden usar los siguientes componentes:
    • tr: Define una fila de la tabla.
    • th: Define una celda de encabezado dentro de una fila.
    • td: Define una celda de datos dentro de una fila.
    • colspan: Atributo que permite que una celda se extienda a través de varias columnas.
    • thead: Define el encabezado de la tabla.
    • tbody: Define el cuerpo de la tabla.
    • tfoot: Define el pie de la tabla.
    • caption: Define un título para la tabla.

  12. ¿Cuáles son los principales controles de una forma HTML5?
    13. Algunos de los principales controles de un formulario HTML5 incluyen:
    • input: Permite la entrada de datos, con diferentes tipos como text, email, password, entre otros.
    • textarea: Permite la entrada de texto multilínea.
    • select: Permite seleccionar una opción de una lista desplegable.
    • button: Permite crear botones para enviar el formulario o realizar otras acciones.
    • label: Proporciona etiquetas para los controles del formulario.
    • fieldset y legend: Permiten agrupar controles relacionados dentro de un formulario.
  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

  14. Sobre el ciclo de vida y desarrollo de los sistemas de información:
      ¿Cuál es el ciclo de vida de los sistemas de información?
    El ciclo de vida de los sistemas de información es el proceso que sigue un sistema de información desde su concepción hasta su retiro. Las etapas principales del ciclo de vida son:
    1. Iniciación
    2. Desarrollo
    3. Operación
    4. Retiro
      ¿Cuál es el ciclo de desarrollo de sistemas de información?
    El ciclo de vida del desarrollo de sistemas de información, o SDLC (por sus siglas en inglés, Software Development Life Cycle) es un proceso estructurado que se sigue para desarrollar, implementar y mantener un sistema de información. Las etapas principales del ciclo de vida son:
    1. Análisis de sistema: Se identifican las necesidades del negocio y se define el alcance del proyecto.
    2. Diseño: Se crea la arquitectura del sistema y se diseñan los componentes.
    3. Desarrollo: Se escribe el código y se construye el sistema.
    4. Pruebas: Se realizan pruebas para asegurar la calidad y funcionalidad del sistema.
    5. Despliegue: Se despliega el sistema en el entorno de producción.
    6. Mantenimiento: Se realizan actualizaciones, correcciones de errores y mejoras continuas al sistema.
